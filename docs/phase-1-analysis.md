# Phase 1: System Analysis and Functional Design

## Project Context

ระบบบริหารลูกหนี้เงินยืม สำหรับมหาวิทยาลัยเทคโนโลยีราชมงคลอีสาน วิทยาเขตสุรินทร์ เป็น Web Application สำหรับควบคุมกระบวนการขอยืมเงิน การตรวจสอบ การอนุมัติ การจ่ายเงิน การส่งใช้เงินยืม การขอขยายอายุสัญญา การออกหนังสือทวงเงิน และการจัดทำรายงานสำหรับผู้บริหาร

ระบบออกแบบจากตัวอย่างคู่มือ `manual_loan_sys.pdf` โดยปรับบริบทหน่วยงานและบทบาทให้เหมาะกับการใช้งานในวิทยาเขตสุรินทร์

## Primary Goals

- ลดงานเอกสารและการติดตามรายการเงินยืมด้วยมือ
- ให้ผู้ยืมตรวจสอบสถานะคำขอและภาระหนี้ของตนเองได้
- ให้เจ้าหน้าที่การเงินและงบประมาณตรวจสอบรายการตามลำดับงานที่ชัดเจน
- รองรับการพิมพ์เอกสารสัญญายืม หนังสือขยายอายุ หนังสือทวง และรายงาน
- เก็บประวัติการเปลี่ยนสถานะและการดำเนินการทุกขั้นตอนเพื่อการตรวจสอบย้อนหลัง
- สร้าง Dashboard สำหรับผู้บริหารเพื่อติดตามลูกหนี้เงินยืมคงค้างและรายการครบกำหนด

## Organization Scope

ข้อมูลหน่วยงานควรรองรับโครงสร้างภายในมหาวิทยาลัย เช่น

- สำนักงานวิทยาเขตสุรินทร์
- คณะ/สำนัก/สถาบัน
- สาขาวิชา/งาน/แผนก
- กองคลังหรือหน่วยงานการเงินที่รับผิดชอบ

## Core Modules

### 1. Authentication and User Management

- สมัครสมาชิก
- เข้าสู่ระบบ
- ลืมรหัสผ่าน
- จัดการข้อมูลโปรไฟล์
- จัดการสิทธิ์ตามบทบาท
- รองรับ JWT และ NextAuth.js

### 2. Loan Request Management

- บันทึกรายการขอยืมเงิน
- ค้นหารายการจากเลขที่คำขอ ชื่อ นามสกุล เลขที่สัญญา หรือหน่วยงาน
- ดูรายละเอียดรายการขอยืม
- พิมพ์สัญญายืมเงิน
- Export เป็น PDF และ Word
- แนบไฟล์ประกอบคำขอ

### 3. Review and Approval Workflow

- ส่วนงานตรวจสอบรายการขอยืม
- งบประมาณตรวจสอบรายการขอยืม
- การเงินตรวจสอบรายการขอยืม
- ผู้บริหารหรือผู้มีอำนาจอนุมัติรายการ
- บันทึกหมายเหตุการตรวจสอบ
- บันทึกประวัติการเปลี่ยนสถานะ

### 4. Payment Management

- บันทึกวิธีจ่ายเงิน
- บันทึกวันที่จ่ายเงิน
- บันทึกข้อมูลเพิ่มเติมกรณียืมจากมหาวิทยาลัย เช่น แผนงาน กองทุน และเลขที่ใบรายละเอียดการจ่าย
- ปรับสถานะรายการเป็นจ่ายแล้ว

### 5. Loan Repayment Management

- บันทึกการส่งใช้เงินยืม
- รองรับการส่งใช้ด้วยเงินสด เช็ค และใบสำคัญ
- บันทึกเลขที่ใบเสร็จ เลขที่เช็ค หรือเลขที่ใบสำคัญจ่าย
- แสดงยอดยืม ยอดส่งใช้แล้ว และยอดคงเหลือ
- ปิดรายการเมื่อส่งใช้ครบ

### 6. Contract Extension

- ผู้ยืมยื่นคำขอขยายอายุสัญญาเงินยืม
- ระบุเหตุผลและรายละเอียดสำหรับบันทึกข้อความ
- เจ้าหน้าที่การเงินตรวจสอบและปรับสถานะการขอขยาย
- บันทึกวันที่ครบกำหนดใหม่เมื่ออนุมัติ

### 7. Debt Notice Letter

- สร้างบันทึกข้อความทวงเงินยืม
- เลือกเลขที่สัญญาที่ต้องการทวง
- ระบุส่วนราชการ เลขที่หนังสือ วันที่ และรายละเอียดหนังสือ
- พิมพ์หรือ Export PDF/Word

### 8. Reports and Dashboard

- รายงานรายการเงินยืมทั้งหมด
- รายงานลูกหนี้คงค้าง
- รายงานรายการครบกำหนด
- รายงานรายการเกินกำหนด
- รายงานการส่งใช้เงินยืม
- รายงานแยกตามหน่วยงาน
- Dashboard สรุปยอดเงินยืม ยอดคงค้าง จำนวนรายการรอตรวจสอบ และรายการเกินกำหนด

## Main Workflow

### Workflow A: Request Loan

1. ผู้ยืมเข้าสู่ระบบ
2. เปิดเมนูบันทึกรายการขอยืม
3. เพิ่มรายการขอยืมและกรอกรายละเอียด
4. ระบบสร้างเลขที่คำขอและสถานะ `pending_department_review`
5. ผู้ยืมพิมพ์สัญญายืมหรือ Export PDF/Word ได้

### Workflow B: Review Loan Request

1. เจ้าหน้าที่ตรวจสอบส่วนงานเปิดรายการที่รอตรวจสอบ
2. เลือกประเภทการยืม: ยืมจากส่วนงาน หรือยืมจากมหาวิทยาลัย
3. เลือกแหล่งเงิน เช่น เงินทุนหมุนเวียน หรือเงินทดรองจ่าย
4. กำหนดเงื่อนไขวันครบกำหนด
5. ส่งต่อรายการ
6. ถ้ายืมจากมหาวิทยาลัย สถานะเป็น `pending_university_finance_review`
7. ถ้ายืมจากส่วนงาน สถานะเป็น `pending_department_approval`

### Workflow C: Budget and Finance Review

1. เจ้าหน้าที่งบประมาณตรวจสอบรายการ
2. หากผ่าน เปลี่ยนสถานะเป็น `budget_reviewed`
3. เจ้าหน้าที่การเงินตรวจสอบรายการ
4. หากผ่าน เปลี่ยนสถานะเป็น `finance_reviewed`
5. รายการพร้อมเข้าสู่ขั้นตอนอนุมัติ

### Workflow D: Approve and Pay

1. ผู้มีอำนาจหรือเจ้าหน้าที่ที่ได้รับสิทธิ์เปิดรายการ
2. ปรับสถานะเป็น `approved`
3. ระบุวิธีจ่ายเงิน
4. หลังจ่ายเงินจริง ปรับสถานะเป็น `paid`
5. ระบุวันที่จ่ายเงิน
6. กรณียืมจากมหาวิทยาลัย ระบุแผนงาน กองทุน และเลขที่ใบรายละเอียดการจ่าย

### Workflow E: Repay Loan

1. เจ้าหน้าที่การเงินเปิดเมนูบันทึกการส่งใช้เงินยืม
2. ระบบแสดงเฉพาะรายการที่ยังส่งใช้ไม่ครบ
3. เลือกรูปแบบการส่งใช้เงินยืม
4. กรอกจำนวนเงิน เลขที่ใบเสร็จ เลขที่เช็ค หรือเลขที่ใบสำคัญ
5. ระบบคำนวณยอดคงเหลือ
6. หากส่งใช้ครบ เปลี่ยนสถานะเป็น `fully_repaid`
7. หากยังไม่ครบ เปลี่ยนสถานะเป็น `partially_repaid`

### Workflow F: Extend Contract

1. ผู้ยืมเลือกสัญญาที่ต้องการขอขยายอายุ
2. กรอกเหตุผลและรายละเอียดบันทึกข้อความ
3. ระบบบันทึกสถานะ `extension_requested`
4. เจ้าหน้าที่การเงินตรวจสอบคำขอ
5. หากอนุมัติ ระบบบันทึกวันครบกำหนดใหม่และสถานะ `extension_approved`
6. หากไม่อนุมัติ ระบบบันทึกสถานะ `extension_rejected`

### Workflow G: Debt Notice

1. เจ้าหน้าที่การเงินเปิดเมนูบันทึกข้อความทวงเงินยืม
2. เลือกเลขที่สัญญาที่ค้างชำระหรือเกินกำหนด
3. ระบุรายละเอียดหนังสือ
4. ระบบบันทึกหนังสือทวงและให้พิมพ์หรือ Export PDF/Word

## Business Rules

- ผู้ยืมสามารถแก้ไขคำขอได้เฉพาะสถานะ `draft` หรือรายการที่ถูกตีกลับ
- รายการที่ส่งตรวจสอบแล้วต้องเปลี่ยนสถานะผ่าน workflow ที่กำหนด
- การบันทึกจ่ายเงินทำได้เฉพาะรายการที่อนุมัติแล้ว
- การบันทึกส่งใช้เงินยืมทำได้เฉพาะรายการที่จ่ายแล้วหรือส่งใช้บางส่วน
- ยอดส่งใช้รวมต้องไม่เกินยอดเงินยืม
- หากยอดส่งใช้ครบ ระบบต้องปิดรายการเป็น `fully_repaid`
- หากพ้นวันครบกำหนดและยังส่งใช้ไม่ครบ ระบบควรแสดงเป็นรายการเกินกำหนด
- การขอขยายอายุต้องมีเหตุผลและวันครบกำหนดใหม่
- การเปลี่ยนสถานะทุกครั้งต้องบันทึก Audit Log
- การลบข้อมูลสำคัญควรใช้ Soft Delete เพื่อรักษาประวัติ

## Dashboard Requirements

### Borrower Dashboard

- รายการคำขอของตนเอง
- รายการรอตรวจสอบ
- รายการจ่ายแล้ว
- รายการที่ยังส่งใช้ไม่ครบ
- รายการครบกำหนดเร็วๆ นี้

### Finance Dashboard

- รายการรอตรวจสอบการเงิน
- รายการรอบันทึกจ่ายเงิน
- รายการยังส่งใช้ไม่ครบ
- รายการเกินกำหนด
- หนังสือทวงที่ออกแล้ว

### Executive Dashboard

- ยอดเงินยืมรวม
- ยอดลูกหนี้คงค้าง
- จำนวนรายการเกินกำหนด
- สรุปตามหน่วยงาน
- แนวโน้มการยืมและการส่งใช้รายเดือน

### Admin Dashboard

- จำนวนผู้ใช้
- จำนวนหน่วยงาน
- รายการกิจกรรมล่าสุด
- สถานะระบบและข้อมูลตั้งต้น

## Report Requirements

- รายงานรายการขอยืมเงินทั้งหมด
- รายงานสัญญาเงินยืมตามช่วงวันที่
- รายงานลูกหนี้คงค้าง
- รายงานรายการครบกำหนด
- รายงานรายการเกินกำหนด
- รายงานการส่งใช้เงินยืม
- รายงานตามหน่วยงาน
- รายงานสำหรับผู้บริหารแบบสรุป

## Document Output Requirements

- สัญญายืมเงิน
- ใบรายละเอียดการจ่ายเงิน
- บันทึกข้อความขอขยายอายุสัญญาเงินยืม
- บันทึกข้อความทวงเงินยืม
- รายงาน PDF
- Export Word สำหรับเอกสารราชการที่ต้องแก้ไขต่อ

## Non-Functional Requirements

- รองรับ Responsive UI สำหรับ Desktop, Tablet และ Mobile
- ใช้ Role-Based Access Control
- ใช้ JWT Session และ Refresh Token ตามความเหมาะสม
- เข้ารหัสรหัสผ่านด้วย bcrypt
- ตรวจสอบข้อมูลทั้งฝั่ง Frontend และ Backend
- รองรับ Pagination, Search และ Filter ในหน้ารายการ
- รองรับ Audit Log สำหรับการเปลี่ยนแปลงข้อมูลสำคัญ
- รองรับ Export PDF/Word
- รองรับภาษาไทยเต็มรูปแบบ และจัดรูปแบบวันที่/เงินแบบไทย

## Open Decisions for Next Phases

- จะใช้ Next.js แบบ App Router รวม API บางส่วน หรือแยก Backend Express เต็มรูปแบบ
- จะใช้ NextAuth.js เชื่อม Google/Microsoft SSO หรือใช้ Email/Password เป็นหลักก่อน
- รูปแบบเลขที่คำขอและเลขที่สัญญาของวิทยาเขตสุรินทร์ต้องกำหนดจากระเบียบจริง
- รายละเอียดแบบฟอร์มราชการควรอ้างอิง template จริงของมหาวิทยาลัย
- ต้องกำหนดรายการหน่วยงานตั้งต้นสำหรับ seed data
- ต้องกำหนดเงื่อนไขวันครบกำหนดมาตรฐาน เช่น กี่วันหลังรับเงิน หรือกี่วันหลังสิ้นสุดโครงการ

## Phase 1 Acceptance Criteria

- ระบุบทบาทผู้ใช้ครบถ้วน
- ระบุโมดูลหลักตามคู่มือระบบตัวอย่างครบถ้วน
- ระบุ workflow การขอยืม ตรวจสอบ อนุมัติ จ่ายเงิน ส่งใช้ ขยายอายุ และทวงเงินครบถ้วน
- ระบุสถานะเอกสารและ lifecycle สำหรับใช้ต่อในการออกแบบฐานข้อมูล
- ระบุ business rules เบื้องต้นสำหรับ Backend และ Frontend
- ระบุ dashboard, report และ document output ที่ต้องพัฒนาต่อใน Phase ถัดไป
